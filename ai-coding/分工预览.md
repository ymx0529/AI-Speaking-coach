模块职责一句话

- Dev A / Conversation（对话链路）：把"用户按住说话 → AI 语音回复"端到端跑通；含场景选择 UI、发音评分展示、Azure 语音集成、DeepSeek 对话生成。

- Dev B / Coach（教练评测）：挂在对话产出之上；含每轮语法/表达纠错（WS 实时推送）、会话数据累积、课后四维评分报告（HTTP 接口）。

---

为什么这样切就不会冲突

1. 垂直切模块，不切层 — 每人独占一个 `modules/` 目录，跨前后端，两人文件集合交集为空。
2. 中心文件 Phase 0 冻结 — `main.py` / `App.vue` / `store.ts` / `ws.ts` / 依赖清单全部提前写死；联调阶段不再补骨架。
3. 注册表代替改中心文件 — 加功能 = 新增自己模块的文件 + 调 `register_*()`，永不碰中心文件。
4. EventBus 解耦 — Dev B 通过进程内事件总线订阅 Dev A 的 turn 事件，无需直接 import Dev A 代码。

---

主用法（Demo 故事线）

用户打开页面 → 选择"求职面试"场景 → 按住空格键说英语 → 松开 → 看到 Azure 发音评分（准确度/流利度/完整度 + word 高亮）→ AI 语音回复下一个问题 → 右侧出现语法纠错提示 → 结束对话 → 看到课后总结评分报告

---

范围说明（避免过度承诺）

- 本期交付是“两人两天可演示 Demo”，交互形态是“按住说话、松手返回结果”的低延迟回合制练习。
- 题目中的“实时语音对话”在本版本中解释为：单轮内音频实时上传、整轮结束后快速返回 STT / 发音评分 / AI 回复 / 纠错，而不是全双工语音连续打断。
- 若后续要冲更高完成度，再扩展为流式 ASR、边听边说、可打断式 TTS。
